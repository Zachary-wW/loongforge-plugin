    **check_budget** function:
    ```python
    def check_budget(
        budget: LoopBudget,
        phase_attempts: int,
        total_attempts: int,
        run_start_time: str,
    ) -> ExitReason | None:
        """Return ExitReason if any budget axis is breached, else None.
        IMPORTANT: This check MUST happen before processing validator results.
        If budget is breached, exit reason is ALWAYS EXHAUSTED, never passed (Pitfall 2)."""
        if phase_attempts >= budget.max_attempts_per_phase:
            return ExitReason.EXHAUSTED
        if total_attempts >= budget.max_attempts_per_run:
            return ExitReason.EXHAUSTED
        from datetime import datetime, timezone
        start = datetime.fromisoformat(run_start_time)
        elapsed = (datetime.now(timezone.utc) - start).total_seconds() / 60.0
        if elapsed >= budget.max_wallclock_minutes:
            return ExitReason.EXHAUSTED
        return None
    ```

    **Main function (partial -- states PROBE through ISSUE):**

    - `run_phase_loop(run_dir: Path, phase: int, gh: GhClient, budget: LoopBudget, dry_run: bool = False, max_iterations: int = 100, repos_info: dict | None = None) -> ExitReason`:
      Re-entrant controller implementing LOOP-01 FSM dispatch. The `repos_info` dict (default None) contains `loongforge_repo`, `loongforge_base_ref`, `run_id`. When None, PR/issue steps are skipped (local-only mode).

      1. Read state: `state = LoopState.from_disk(run_dir, phase)`
      2. Record validator_hash at first entry if not recorded.
      3. Budget pre-check (LOOP-03, Pitfall 2). If breached: _transition to EXIT, append budget_check row, persist, return EXHAUSTED.
      4. Safety iteration limit: if max_iterations <= 0, return EXHAUSTED.
      5. FSM dispatch via match/case on state.current_state:
         - `FSMState.PROBE`: _transition to EDIT with kind="probe". Persist. Recurse with max_iterations-1.
         - `FSMState.EDIT`: _transition to PR with kind="edit". Persist. Recurse.
         - `FSMState.PR`:
           - If repos_info: create branch + open PR, store pr_number.
           - _transition to MERGE_BASE with kind="pr", pr_url from result. Persist. Recurse.
         - `FSMState.MERGE_BASE`:
           - If repos_info and state.pr_number: merge PR.
           - _transition to VALIDATE with kind="merge_base". Persist. Recurse.
         - `FSMState.VALIDATE`:
           - Call `run_validator(run_dir, phase, gh)`. Get ValidatorResult.
           - Store validator result in state.last_validator_summary as: `{"status": result.status, "name": result.name, "failure_signature": {"kind": sig.kind, "location": sig.location, "expected": sig.expected, "actual": sig.actual} if result.failure_signature else None}`.
           - IMPORTANT: Budget check FIRST (Pitfall 2). If breached, transition to EXIT with EXHAUSTED.
           - If result.status == "passed": set exit_reason=VALIDATOR_PASSED, _transition to EXIT with kind="validate", verdict="passed", validator=result.name. Persist. Return VALIDATOR_PASSED.
           - If should_rerun_for_flake(result, phase) and result.rerun_count < DEFAULT_FLAKE_RERUN_COUNT: _transition to RERUN with kind="validate_rerun", verdict="failed". SAME attempt number (no _advance_attempt). Persist. Recurse.
           - Otherwise: _transition to DIAGNOSE with kind="validate", verdict="failed", validator=result.name. Persist. Recurse.
         - `FSMState.DIAGNOSE`:
           - Reconstruct a ValidatorResult from state.last_validator_summary (the dict stored in loop_state.yml). Create a minimal ValidatorResult with status, name, failure_signature reconstructed from the summary dict, evidence={}, integrity_ok=True, integrity_details={}.
           - Read attempts_history via _read_attempts_history.
           - Call `classify_failure(reconstructed_result, attempts_history)`. Get DiagnoseResult.
           - If classification is WRONG_DIRECTION or NEEDS_HUMAN: call `write_escalation(run_dir, phase, diagnosis.classification, diagnosis.rationale, attempts_history)`. Set exit_reason=HUMAN_NEEDED. _transition to EXIT with kind="diagnose", exit_reason="human_needed". Persist. Return HUMAN_NEEDED.
           - If CODE_BUG or FLAKY: _transition to ISSUE with kind="diagnose", verdict=diagnosis.classification.value. Persist. Recurse.
         - `FSMState.ISSUE`:
           - If repos_info: call `gh.open_issue(...)` with failure_signature from state.last_validator_summary.
           - Parse issue number from GhResult. Store issue_number.
           - _transition to FIX_PR with kind="issue". Persist. Recurse.
         - `FSMState.EXIT`: persist state. Return state.exit_reason.
