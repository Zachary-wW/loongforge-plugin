<objective>
Build the FSM loop controller that drives Probe -> Edit -> PR -> Merge(base) -> Validate -> (Diagnose -> Issue -> Fix-PR -> Review -> Merge -> Rerun)* per phase with hard budgets, state persistence, and structured logging.

Purpose: This is the FSM spine -- the re-entrant controller that composes the validator wrapper (Plan 01) and GhClient (Phase 2) into a working closed-loop system. It exits only on a verifiable validator-pass or a bounded escalation, never on hope (P3, P18). Critically, on exit it writes phaseN_output.yml with validator_integrity, loop, pr, and issues blocks so that validate_phase_completion (VAL-04 hook from Plan 01 Task 3) can verify the exit.

Output: One new module (loop_controller.py) and one test file.

    **check_budget** function:
    ```python
    def check_budget(
        budget: LoopBudget,
        phase_attempts: int,
        total_attempts: int,
        run_start_time: str,
    ) -> ExitReason | None:
        """Return ExitReason if any budget axis is breached, else None.
        IMPORTANT: This check MUST happen before processing validator results.
        If budget is breached, exit reason is ALWAYS EXHAUSTED, never passed (Pitfall 2)."""
        if phase_attempts >= budget.max_attempts_per_phase:
            return ExitReason.EXHAUSTED
        if total_attempts >= budget.max_attempts_per_run:
            return ExitReason.EXHAUSTED
        from datetime import datetime, timezone
        start = datetime.fromisoformat(run_start_time)
        elapsed = (datetime.now(timezone.utc) - start).total_seconds() / 60.0
        if elapsed >= budget.max_wallclock_minutes:
            return ExitReason.EXHAUSTED
        return None
    ```

      Compute SHA-256 hash of the validator binary for VAL-04 integrity checking.
      ```python
      import hashlib
      def _compute_validator_hash(run_dir: Path) -> str | None:
          """Compute sha256[:16] of the loongforge-phase-gate binary, or None if missing."""
          validator_path = run_dir / "bin" / "loongforge-phase-gate"
          if not validator_path.exists():
              return None
          return hashlib.sha256(validator_path.read_bytes()).hexdigest()[:16]
      ```
      This hash is recorded once at loop start and passed as `recorded_hash` to `check_validator_integrity` on every VALIDATE/RERUN invocation.

    - `_write_phase_output(run_dir: Path, phase: int, state: LoopState, validator_result: ValidatorResult | None, budget: LoopBudget) -> None`:
      Write the `phaseN_output.yml` handoff file with validator_integrity, loop, pr, and issues blocks. This is the BLOCKER fix -- without this write, Plan 01's VAL-04 hook in `_validate_loop_evidence` always rejects passed exits because `validator_integrity` is missing from the file.

      ```python
      import yaml
      from skills.adapt.lib.schema import LoopBlockOutput, PrBlockOutput, IssuesBlockOutput

      def _write_phase_output(
          run_dir: Path, phase: int, state: LoopState,
          validator_result: ValidatorResult | None, budget: LoopBudget,
      ) -> None:
          """Write phaseN_output.yml with loop, validator_integrity, pr, issues blocks.

          This file is read by validate_phase_completion.py -> _validate_loop_evidence,
          which enforces VAL-04: if exit_reason is validator_passed, validator_integrity
          must be present and integrity_ok=True.
          """
          output_path = run_dir / "phases" / f"phase{phase}_output.yml"
          # Read existing file if present (merge with any pre-existing fields)
          existing: dict = {}
          if output_path.exists():
              try:
                  existing = yaml.safe_load(output_path.read_text()) or {}
              except Exception:
                  existing = {}

          # loop block (LoopBlockOutput schema)
          loop_block = LoopBlockOutput(
              attempts=state.attempt,
              max_attempts=budget.max_attempts_per_phase,
              exit_reason=state.exit_reason.value if state.exit_reason else "validator_passed",
              attempts_journal=str(run_dir / "phases" / f"phase{phase}" / "attempts.jsonl"),
          ).model_dump()

          # validator_integrity block (consumed by _validate_loop_evidence VAL-04 hook)
          validator_integrity: dict = {}
          if validator_result is not None:
              validator_integrity = {
                  "integrity_ok": validator_result.integrity_ok,
                  "binary_hash_ok": validator_result.integrity_details.get("binary_hash_ok", True),
                  "log_mtime_ok": validator_result.integrity_details.get("log_mtime_ok", True),
                  "log_present": validator_result.integrity_details.get("log_present", True),
              }

          # pr block
          pr_block = PrBlockOutput(number=state.pr_number).model_dump(exclude_none=True)

          # issues block
          issues_block = IssuesBlockOutput(
              opened=state.issues_opened,
              closed=state.issues_closed,
          ).model_dump()

          # Merge into existing data (existing wins for non-loop keys like status, validator, steps)
          merged = {
              **existing,
              "loop_engineering": True,
              "loop": loop_block,
              "validator_integrity": validator_integrity,
              "pr": pr_block,
              "issues": issues_block,
          }

          output_path.parent.mkdir(parents=True, exist_ok=True)
          output_path.write_text(yaml.dump(merged, sort_keys=False, default_flow_style=False))
      ```

    **Main function (partial -- states PROBE through ISSUE):**

    - `run_phase_loop(run_dir: Path, phase: int, gh: GhClient, budget: LoopBudget, dry_run: bool = False, max_iterations: int = 100, repos_info: dict | None = None) -> ExitReason`:
      Re-entrant controller implementing LOOP-01 FSM dispatch. The `repos_info` dict (default None) contains `loongforge_repo`, `loongforge_base_ref`, `megatron_repo`, `megatron_ref`, `run_id`. When None, PR/issue steps are skipped (local-only mode).

      1. Read state: `state = LoopState.from_disk(run_dir, phase)`
      2. Record validator_hash at first entry if not recorded: `if state.validator_hash is None: state.validator_hash = _compute_validator_hash(run_dir); state.persist(run_dir)`.
      3. Budget pre-check (LOOP-03, Pitfall 2). If breached: _transition to EXIT, append budget_check row, persist, return EXHAUSTED.
      4. Safety iteration limit: if max_iterations <= 0, return EXHAUSTED.
      5. FSM dispatch via match/case on state.current_state:
         - `FSMState.PROBE`: _transition to EDIT with kind="probe". Persist. Recurse with max_iterations-1.
         - `FSMState.EDIT`: _transition to PR with kind="edit". Persist. Recurse.
         - `FSMState.PR`:
           - If repos_info: create branch + open PR, store pr_number.
           - _transition to MERGE_BASE with kind="pr", pr_url from result. Persist. Recurse.
         - `FSMState.MERGE_BASE`:
           - If repos_info and state.pr_number: merge PR.
           - _transition to VALIDATE with kind="merge_base". Persist. Recurse.
         - `FSMState.VALIDATE`:
           - Extract megatron args from repos_info: `megatron_repo = repos_info.get("megatron_repo") if repos_info else None`, `megatron_ref = repos_info.get("megatron_ref") if repos_info else None`.
           - Call `run_validator(run_dir, phase, gh, megatron_repo=megatron_repo, megatron_ref=megatron_ref)`. Get ValidatorResult.
           - If validator_result.loong_megatron_sha is not None: update `state.loong_megatron_sha = validator_result.loong_megatron_sha`.
           - Call `check_validator_integrity(run_dir, phase, state.run_start_time, recorded_hash=state.validator_hash)` to get integrity dict. Update validator_result.integrity_ok and integrity_details from this call (if the wrapper didn't already call it, this ensures the hash check uses the recorded_hash from loop start).
           - Store validator result in state.last_validator_summary as: `{"status": result.status, "name": result.name, "integrity_ok": result.integrity_ok, "integrity_details": result.integrity_details, "failure_signature": {"kind": sig.kind, "location": sig.location, "expected": sig.expected, "actual": sig.actual} if result.failure_signature else None, "loong_megatron_sha": result.loong_megatron_sha}`.
           - IMPORTANT: Budget check FIRST (Pitfall 2). If breached, transition to EXIT with EXHAUSTED.
           - If result.status == "passed": set exit_reason=VALIDATOR_PASSED, _transition to EXIT with kind="validate", verdict="passed", validator=result.name. Call `_write_phase_output(run_dir, phase, state, result, budget)`. Persist. Return VALIDATOR_PASSED.
           - If should_rerun_for_flake(result, phase) and result.rerun_count < DEFAULT_FLAKE_RERUN_COUNT: _transition to RERUN with kind="validate_rerun", verdict="failed". SAME attempt number (no _advance_attempt). Persist. Recurse.
           - Otherwise: _transition to DIAGNOSE with kind="validate", verdict="failed", validator=result.name. Persist. Recurse.
         - `FSMState.DIAGNOSE`:
           - Reconstruct a ValidatorResult from state.last_validator_summary (the dict stored in loop_state.yml). Create a minimal ValidatorResult with status, name, failure_signature reconstructed from the summary dict, evidence={}, integrity_ok from summary, integrity_details from summary, loong_megatron_sha from summary.
           - Read attempts_history via _read_attempts_history.
           - Call `classify_failure(reconstructed_result, attempts_history)`. Get DiagnoseResult.
           - If classification is WRONG_DIRECTION or NEEDS_HUMAN: call `write_escalation(run_dir, phase, diagnosis.classification, diagnosis.rationale, attempts_history)`. Set exit_reason=HUMAN_NEEDED. _transition to EXIT with kind="diagnose", exit_reason="human_needed". Call `_write_phase_output(run_dir, phase, state, None, budget)`. Persist. Return HUMAN_NEEDED.
           - If CODE_BUG or FLAKY: _transition to ISSUE with kind="diagnose", verdict=diagnosis.classification.value. Persist. Recurse.
         - `FSMState.ISSUE`:
           - If repos_info: call `gh.open_issue(...)` with failure_signature from state.last_validator_summary.
           - Parse issue number from GhResult. Store issue_number. Append to state.issues_opened.
           - _transition to FIX_PR with kind="issue". Persist. Recurse.
         - `FSMState.EXIT`: persist state. Return state.exit_reason.
