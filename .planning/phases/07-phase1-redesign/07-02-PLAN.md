Create a new file skills/adapt/references/phases/phase1/perf_rules.yaml with 8 performance guard rail rules (P1-P8). Each rule follows the SAME format as existing structural_rules in strategy_rules.yaml: id, name, applies_to_gaps, when, rule, violation_signal, rationale. Additionally, each rule has violation_severity: blocking (not warning) per D-03.

The 8 rules cover these observed anti-patterns:

P1 (core_attention_no_identity_op): When model uses novel attention variant, core_attention slot MUST be filled with the novel module, NEVER IdentityOp. Violation: core_attention=IdentityOp in submodules spec. Blocking because zero attention output wastes all upstream computation.

P2 (flat_parameter_not_custom_module): When model uses grouped output projection, MUST use flat nn.Parameter (2D) with view() + einsum, NOT custom nn.Module with bmm. Violation: custom GroupedLinear(nn.Module). Blocking because TP splitting expects flat 2D parameters.

P3 (reuse_megatron_moe_infrastructure): When model uses MoE features that Megatron provides, prefer reuse_megatron/wrap_megatron over new_impl. Violation: new_impl for routing/expert MLP/KV projection when config-driven strategy exists. Blocking because custom MoE duplicates parallelism logic with subtle bugs.

P4 (activation_checkpointing_for_large_models): When model has num_layers >= 32 OR ffn_hidden_size >= 65536 OR MoE with num_experts >= 8, MUST include activation checkpointing configuration. Violation: no --checkpoint-activations in example script and no config.activations_checkpoint_method. Blocking because OOM without checkpointing.

P5 (correct_tp_ep_communication): MoE communication MUST use Megatron TokenDispatcher, NOT raw torch.distributed.all_to_all. TP MUST use ColumnParallelLinear/RowParallelLinear via BackendSpecProvider. Violation: raw torch.distributed calls. Blocking because manual comm bypasses overlap-optimization.

P6 (fused_kernel_when_available): When operations have fused kernels (SwiGLU, RMSNorm, RoPE, cross-entropy), MUST use fused paths via Megatron/TE. Violation: manual torch.nn.functional.silu + clamp or manual RMSNorm with torch.rsqrt. Blocking because unfused is 2-5x slower.

P7 (inverse_rope_for_mla_hybrid): When model uses MLA with hybrid attention (qk_nope_head_dim != 0 and qk_rope_head_dim != 0), attention forward MUST implement inverse RoPE. Violation: no inverse_rope method when nope dimensions > 0 and rope dimensions > 0. Blocking because mixed rotary/non-rotary dimensions produce incorrect gradients.

P8 (activation_offload_context): When model has num_layers >= 64 OR MoE with num_experts >= 64, MUST support activation offloading context. Violation: no offloading config for 64+ experts. Blocking because OOM without offloading.

Version header: version: 1, purpose field describing the anti-pattern prevention goal.
