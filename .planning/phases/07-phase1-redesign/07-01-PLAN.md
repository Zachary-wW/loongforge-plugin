Replace the current Step 2 branch structure with a confidence-driven 3-level validation depth system:
- Branch A: diff == same (unchanged) -> reuse_ref, no Megatron reading needed.

  - 2b. Find candidate Omni ModuleSpec (unchanged)

    - confidence=high: SKIP 2c entirely. Adopt bridge_mapping strategy directly. Verify reference_impl_analysis.yaml entry exists and is complete.
    - confidence=medium: SIMPLIFIED 2c. Only read Megatron source sections related to behavioral_diff topics. May override strategy with evidence.

    - gap (megatron=null): NO 2c. Read bridge_mapping.gaps[].phase1_guidance. Design new module in Step 2d.
  - 2d. MEGATRON GAP MODULE DESIGN (new step):
    When a component has megatron=null (gap in bridge_mapping), Step 2d designs the new Megatron module: reads phase1_guidance, reads HF source for the gap component, determines Megatron integration point (new file or existing-file modification), designs module interface (class name, init, forward, submodule slots, config fields), records design in strategy_plan.

**4. Step 3 -- Per-File Code Generation with dual-repo output (per D-01, D-02):**
- Generate LoongForge files (existing flow unchanged per FILE_STRUCTURE.md order)
- Generate Megatron files (NEW): For each gap module designed in Step 2d, create new .py file. Verify Megatron files import correctly.

- Step 3 reads perf_rules.yaml (references/phases/phase1/perf_rules.yaml) alongside strategy_rules.yaml. P1-P8 rules are enforced as blocking errors parallel to G1-G14 (per D-03, D-04).

**5. Loop Engineering Hooks -- dual-repo (per D-01):**
- Pre-Edit: Create branch on BOTH repos (loongforge + megatron).
- Post-Edit: Create PR on BOTH repos. Megatron PR body pins LoongForge commit SHA (VAL-05). Merge BOTH PRs before validator runs.

**6. FSM Exit Path -- explicitly described (per D-08):**
Add new section "## Loop FSM Exit Path" with two scenarios:
- When repos: present: Step 3 generates code -> commits to branches on both repos -> PRs created and merged -> validator runs -> on pass: done -> on fail: exit to loop_controller (VALIDATE -> DIAGNOSE -> ISSUE -> FIX_PR -> REVIEW -> MERGE_FIX -> RERUN).
