  <action>
    Create `skills/adapt/lib/idempotency.py` with the following exports:

    1. `IdempotencyFooter` -- a `NamedTuple` with fields `run_id: str`, `phase: int`, `attempt: int`, `action: str`, `key: str`

    2. `compute_idempotency_key(run_id: str, phase: int, attempt: int, action_kind: str) -> str`
       - Implementation: `hashlib.sha256(f"{run_id}:{phase}:{attempt}:{action_kind}".encode()).hexdigest()`
       - Returns 64-char lowercase hex string
       - This is the RESUME-03 contract: same inputs always produce the same key, enabling crash-resume dedup

    3. `format_footer(run_id: str, phase: int, attempt: int, action_kind: str) -> str`
       - Implementation: compute key, then return the string:
         `"\n<!-- adapt-skill: run={run_id} phase={phase} attempt={attempt} action={action_kind} key={key} -->\n"`
       - The footer is an HTML comment so it is invisible in rendered Markdown but parseable by machines
       - ALSO include a visible machine-readable fallback line immediately before the HTML comment:
         `[adapt-skill-key: {key}]`
       - This fallback addresses the open question from RESEARCH about whether GitHub search indexes HTML comments.
         The visible line ensures searchability even if HTML comments are not indexed.

    4. `parse_footer(body: str) -> Optional[IdempotencyFooter]`
       - Search body for the HTML comment pattern: `<!-- adapt-skill: run=(\S+) phase=(\d+) attempt=(\d+) action=(\S+) key=([a-f0-9]{64}) -->`
       - If found, return IdempotencyFooter with extracted fields
       - If not found, return None
       - Use `re.search` with the pattern; the regex must capture all 5 fields

    Create `skills/adapt/tests/lib/test_idempotency.py` with pytest tests covering all 9 behaviors listed above.
    Use `from skills.adapt.lib.idempotency import compute_idempotency_key, format_footer, parse_footer, IdempotencyFooter`.
  </action>

    2. `def pr_title(run_id: str, phase: int, attempt: int, validator: str = "", kind: str = "base") -> str`
       - Template: `[adapt][{kind}][run-{run_id}] phase-{phase} attempt-{attempt}{validator_suffix}`
       - Where `validator_suffix = f" ({validator})" if validator else ""`
       - `kind` is one of "base", "fix", "diagnose" -- appears in brackets
       - Example output: `[adapt][base][run-abc123] phase-2 attempt-1 (phase2-conversion)`
       - This addresses PR-03: title contains run_id, phase, attempt, validator name

    3. `def pr_body(run_id: str, phase: int, attempt: int, kind: str = "base", validator: str = "", diff_summary: str = "", fixes_issue: Optional[int] = None) -> str`
       - Build the body from these sections:
         a. `## Summary` -- one-liner: `Adapt skill {kind} PR for phase {phase}, attempt {attempt}.`
         b. `## Validator` -- `{validator}` if non-empty, else "N/A (base PR)"
         c. `## Diff Summary` -- `{diff_summary}` if non-empty, else "(see PR diff)"
         d. `## Linked Issue` -- `Fixes #{fixes_issue}` if fixes_issue is not None, else "N/A"
       - Append idempotency footer via `format_footer(run_id, phase, attempt, f"{kind}-pr")`
       - The `Fixes #N` syntax is exact per GitHub recognition rules (ISSUE-02)

    4. `def issue_title(phase: int, validator_name: str, failure_kind: str) -> str`
       - Template: `[adapt] phase-{phase} {validator_name}: {failure_kind}`
       - Example: `[adapt] phase-2 loss-diff: numerical_mismatch`

    5. `def issue_body(phase: int, validator_name: str, failure_signature: dict, log_excerpt: str, attempts_jsonl_link: str, reproduction_cmd: str, run_id: str = "", attempt: int = 0) -> str`
       - Build the body from these sections:
         a. `## Failure Signature` -- structured table:
            ```
            | Field     | Value |
            |-----------|-------|
            | kind      | {failure_signature.get("kind", "")} |
            | location  | {failure_signature.get("location", "")} |
            | expected  | {failure_signature.get("expected", "")} |
            | actual    | {failure_signature.get("actual", "")} |
            ```
         b. `## Log Excerpt` -- `{log_excerpt}` (caller passes last N lines)
         c. `<details><summary>Full Log</summary>` -- placeholder; caller fills from file
         d. `## Attempts Log` -- `[attempts.jsonl]({attempts_jsonl_link})`
         e. `## Reproduction` -- `{reproduction_cmd}` in a code block
       - Append idempotency footer via `format_footer(run_id, phase, attempt, "issue")`
       - This addresses ISSUE-01: structured failure_signature, log excerpt, attempts.jsonl link, reproduction command

    6. `def dedup_comment(attempt: int, log_excerpt: str, timestamp: str) -> str`
       - Template:

         ```
       - This addresses ISSUE-03 / D-02: appended to existing issue instead of creating duplicate

       - This addresses PR-05 / D-01: posts this comment when human commits are detected

         ```
