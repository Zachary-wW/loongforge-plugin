# ckpt_roundtrip -- Weight Roundtrip Verification Tool

## Responsibility

Verify that after the full HF -> mcore -> HF conversion chain, the rebuilt HF checkpoint has no missing keys, unexpected keys, shape mismatches, or tensor value mismatches under the supported precision rule. Before tensor comparison, verify that the rebuilt HF checkpoint was produced from a real target-framework/MCore checkpoint rather than a reversible HF container, metadata-only wrapper, or direct copy from the original HF input.

`ckpt-roundtrip` is the offline rebuilt-HF comparison gate inside the authoritative Phase 2 validator `phase2-conversion`. A failure is repairable by Phase 2 unless it exposes a model-code issue that requires fallback to Phase 1.

## Execution Steps

1. Confirm that `<run_dir>/phases/phase2/convert_output/rebuilt_hf` has been generated by Phase 2 Step 5b, unless the caller explicitly passes a phase-local `run_dir` that already points at `<adaptation_run>/phases/phase2`.
2. Run the production/provenance gate before tensor comparison:
   - MCore/target checkpoint output exists and contains target-framework checkpoint artifacts, not only HF `model.safetensors.index.json` / `pytorch_model.bin.index.json` files.
   - Rebuilt HF provenance/logs show it was generated from the MCore/target checkpoint output.
   - Original HF weight files were not directly copied into rebuilt HF except tokenizer/config/processor ancillary files.
   - No opaque shortcut is detected (`reversible_hf_container`, `metadata_only_roundtrip`, `hf_key_preserving_mcore_stub`, or `rebuilt_hf_from_original_source_without_mcore_load`).
3. Compare the original HF ckpt with the rebuilt HF ckpt:

```python
import json
from pathlib import Path

def load_weight_map(ckpt_dir: str) -> dict:
    """Supports both safetensors / bin index formats"""
    p = Path(ckpt_dir)
    for fname in ("model.safetensors.index.json", "pytorch_model.bin.index.json"):
        if (p / fname).exists():
            return json.loads((p / fname).read_text())["weight_map"]
    raise FileNotFoundError(f"Weight index file not found: {ckpt_dir}")

def roundtrip_check(orig_hf_path: str, rebuilt_hf_path: str) -> dict:
    import torch
    from safetensors import safe_open

    orig_map = load_weight_map(orig_hf_path)
    rebuilt_map = load_weight_map(rebuilt_hf_path)

    orig_keys = set(orig_map.keys())
    rebuilt_keys = set(rebuilt_map.keys())

    missing_keys = sorted(orig_keys - rebuilt_keys)
    unexpected_keys = sorted(rebuilt_keys - orig_keys)
    shape_mismatches = []
    tensor_mismatches = []

    for key in sorted(orig_keys & rebuilt_keys):
        with safe_open(f"{orig_hf_path}/{orig_map[key]}", framework="pt") as f:
            orig_shape = tuple(f.get_slice(key).get_shape())
            orig_tensor = f.get_tensor(key).float()
        with safe_open(f"{rebuilt_hf_path}/{rebuilt_map[key]}", framework="pt") as f:
            rebuilt_shape = tuple(f.get_slice(key).get_shape())
            rebuilt_tensor = f.get_tensor(key).float()
        if orig_shape != rebuilt_shape:
            shape_mismatches.append(f"{key}: {orig_shape} != {rebuilt_shape}")
            continue
        if not torch.allclose(orig_tensor, rebuilt_tensor, atol=1e-2, rtol=1e-2):
            max_diff = (orig_tensor - rebuilt_tensor).abs().max().item()
            tensor_mismatches.append(f"{key}: max_abs_diff={max_diff}")

    passed = not (missing_keys or unexpected_keys or shape_mismatches or tensor_mismatches)
    return {
        "passed": passed,
        "missing_keys": missing_keys,
        "unexpected_keys": unexpected_keys,
        "shape_mismatches": shape_mismatches,
        "tensor_mismatches": tensor_mismatches,
    }
```

## Pass Conditions

- `production_gate.mcore_artifacts_exist` is true when target-framework conversion is required
- `production_gate.loaded_by_target_framework` is true when target-framework loading is required
- `production_gate.rebuilt_hf_derived_from_mcore` is true
- `production_gate.reversible_container_detected` is false
- `production_gate.forbidden_shortcuts` is empty
- `missing_keys` is empty
- `unexpected_keys` is empty
- `shape_mismatches` is empty
- `tensor_mismatches` is empty under the supported precision rule

## Failure Handling

Output `production_gate` first, then specific inconsistencies grouped as missing keys, unexpected keys, shape mismatches, and tensor value mismatches. If production provenance fails, hand off to Phase 2 with `failure_gate="non_production_conversion"` even if tensor values would match. Otherwise hand off to Phase 2 to fix convert YAML, shell arguments, or appended converter code and re-execute the affected conversion chain.
Retry budget and `human_needed` escalation are owned by the caller, normally the Phase 2 `phase2-conversion` loop.
