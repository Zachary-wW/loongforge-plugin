# loss-diff -- Precision Verification and Per-Layer Diagnosis

## Responsibility

Compare LoongForge against a reference framework on real-weight step-1 forward loss and step 2..N train-step losses, then locate divergence layers with tensor dumps when needed. Gradient norms are diagnostic evidence and only fail the gate when they are NaN/Inf.

Phase 3/4 must pass an `omni_script` copied from the generated `examples/<model>/` script. Do not synthesize an independent LoongForge `torchrun` command inside this tool. In reference mode, the copied script must launch through `torchrun`, enter `loongforge/train.py`, export the actual runtime batch and runtime loss, and the reference side must recompute loss on that exported batch on the same effective device class unless a written contract explicitly justifies a different device.

## Prerequisites

1. Phase 2 has passed and `mcore_ckpt` points to the converted checkpoint.
2. `convert_yaml` exists and is the Phase 2 conversion YAML used to map HF names to LoongForge names.
3. `mock_input_path` exists or can be generated by `hf_tensor_dump.py --generate-mock-input`.
4. `omni_script` exists, is copied from the generated Phase 1 example script, loads `mcore_ckpt`, launches through `torchrun`, enters `loongforge/train.py`, exports the actual runtime batch/loss, and writes logs under the current phase directory.
5. When provided, `production_gate` proves the checkpoint is a real target-framework/MCore artifact and not a reversible HF container.
6. Reference-mode verification has explicit evidence for `real_example_script_used`, `torchrun_used`, `entrypoint_verified`, `runtime_batch_export_path`, `runtime_loss_export_path`, `hf_recomputed_on_exported_real_batch`, `same_batch_verified`, and `same_device_verified`.
7. Optional per-layer diagnosis requires Omni tensor logging flags: `--enable-log-tensor`, `--log-tensor-stage`, `--save-tensor`, `--save-tensor-dir`.

## Availability and Contract Gate

Before execution, determine whether the selected reference path is available and whether the Phase 0/2 contracts permit this comparison. If not, degrade only where explicitly allowed or return `human_needed`.

Contract checks:
- If `production_gate` is provided, it must show target checkpoint provenance passed before reference-mode loss diff runs.
- If `reference_contract_path` marks a reference as required for `precision_reference`, the selected `reference_type` must satisfy that requirement.
- If `phase3_reference_requirements.custom_reference_loader_required` is true, `reference_loader` must be provided and must cover the declared behavior/key-transform requirements.
- Standalone mode cannot be used as an automatic downgrade from HF/Megatron reference failure; it must be explicitly requested and contract-permitted.

Write these checks into `verify_report.json.details.contract_preflight`.

| Scenario | Required Conditions | Executable Capabilities | Handling When Not Met |
|----------|---------------------|-------------------------|----------------------|
| Omni vs HF | `hf_tensor_dump.py` can load `hf_path`, `convert_yaml`, and `mock_input_path`; `omni_script` is runnable | step-1 forward loss diff, step 2..N train-step loss diff, grad norm diagnostics, HF tensor dump, Omni tensor dump | If HF dump fails after retry, return `failed` or `human_needed`; do not invent a different HF command |
| Omni vs Megatron-backend | Reference framework can run the same mock input and expose compatible forward loss, train-step losses, grad norm diagnostics, and tensor dumps | step-1 forward loss diff, step 2..N train-step loss diff, optional per-layer tensor diff | If reference tensor dump is unavailable, keep loss/train-step evidence and return `human_needed` only when diagnosis is required |
| Standalone-smoke | `omni_script` can run forward/backward without reference and dispatcher explicitly scoped the run to standalone | loss range, determinism, gradient sanity, weight update | No absolute precision guarantee; record standalone-smoke checks only and do not present as HF/Megatron alignment |

## Four-Stage Execution Flow

### Stage 0: Prepare mock input

If `mock_input_path` is missing, generate it with the actual script interface:

```bash
python references/phases/phase3/scripts/hf_tensor_dump.py \
    --generate-mock-input \
    --seq-len 128 \
    --mock-input-path $mock_input_path
```

### Stage 1: Forward Loss Comparison

1. Run LoongForge through `omni_script`:

```bash
bash $omni_script
```

The caller-prepared script must load `$mcore_ckpt`, use `$mock_input_path`, set `--train-iters 1`, and write logs under `$run_dir`. Extract `loss_omni` from the log.

2. For HF reference, first confirm the caller selected a trustworthy reference loader. If model_spec behavior modifications include FP8 reference loading, checkpoint key transforms, MTP context, or sliced-checkpoint reference notes, plain `AutoModelForCausalLM.from_pretrained()` is not sufficient unless the caller explicitly explains why it is valid.

Default command:

```bash
python references/phases/phase3/scripts/hf_tensor_dump.py \
    --hf-ckpt-path $hf_path \
    --convert-yaml-path $convert_yaml \
    --stage forward \
    --mock-input-path $mock_input_path \
    --save-dir $run_dir/hf_tensors
```

If a custom reference loader is required but not available, return `human_needed` with `failure_gate="reference_loader_required"` and record original HF path, sliced HF path if any, expected FP8/MTP/key handling, and missing/unexpected key evidence. Extract `loss_ref` only after reference loading is trustworthy.

3. Compare `abs(loss_omni - loss_ref) < 1e-3`.

PASS proceeds to Stage 2. FAIL proceeds to Stage 1b.

### Stage 1b: Forward Per-Layer Diagnosis

1. Create a diagnostic copy of `omni_script` under `$run_dir/loss_diff/forward_tensor_dump.sh`.
2. Modify only the copy to add forward tensor logging flags:

```bash
--enable-log-tensor \
--log-tensor-name-pattern ".*layers.*" \
--log-tensor-stage forward \
--log-tensor-iter-pattern "1" \
--log-tensor-rank "0" \
--save-tensor \
--save-tensor-dir $run_dir/omni_tensors
```

3. Compare Omni tensors with HF tensors using a self-devised approach:
   - Load saved tensors from `$run_dir/omni_tensors` and `$run_dir/hf_tensors`
   - For each matching layer tensor pair, compute: max absolute diff, relative L1 norm, cosine similarity
   - Identify the first diverging layer (where relative L1 > 1e-4)
   - Write a summary report to `$run_dir/tensor_compare_forward/diagnosis.json` containing: layer name, metric values, divergence rank

4. Output diagnostic evidence (diverging layer name, metric values, upstream/downstream context) and return `human_needed`.

### Stage 2: Train-Step Loss Comparison

1. Run LoongForge through `omni_script` in training mode for the agreed Phase 3 train-step count and extract `loss_omni_steps`, where step 1 is the already-checked forward loss and steps 2..N are post-update train-step losses.
2. For HF reference, use the same trusted reference loader decision from Stage 1:

```bash
python references/phases/phase3/scripts/hf_tensor_dump.py \
    --hf-ckpt-path $hf_path \
    --convert-yaml-path $convert_yaml \
    --stage forward,backward \
    --mock-input-path $mock_input_path \
    --save-dir $run_dir/hf_tensors_fb
```

If Stage 1 required a custom loader, Stage 2 must use the same loader or return `human_needed`; do not mix reference loading paths between forward-only and forward/backward checks.

3. Extract `loss_ref_steps` and compare every post-update step: `abs(loss_omni_steps[i] - loss_ref_steps[i]) < 1e-3` for each `i >= 2`.
4. Record `grad_norm_omni_steps` and `grad_norm_ref_steps` as diagnostics. NaN/Inf grad norms fail the gate; finite grad norm differences do not fail the gate by themselves.

PASS completes precision verification. FAIL proceeds to Stage 2b.

### Stage 2b: Train-Step / Backward Diagnosis

1. Create a diagnostic copy of `omni_script` under `$run_dir/loss_diff/backward_tensor_dump.sh`.
2. Modify only the copy to add backward tensor logging flags. Omni accepts only one `--log-tensor-stage` value, so backward diagnosis must be a separate run:

```bash
--enable-log-tensor \
--log-tensor-name-pattern ".*layers.*" \
--log-tensor-stage backward \
--log-tensor-iter-pattern "1" \
--log-tensor-rank "0" \
--save-tensor \
--save-tensor-dir $run_dir/omni_tensors_bwd
```

3. Compare with `$run_dir/hf_tensors_fb` using the same self-devised tensor comparison approach as Stage 1b (load tensor pairs, compute max abs diff / relative L1 / cosine sim, identify diverging layers).
4. Output diagnostic evidence and return `human_needed`.

## Standalone-Smoke Mode

Standalone-smoke mode uses only `omni_script` and does not compare against HF/Megatron. It may return top-level `passed` only when the caller explicitly requested `reference_type=standalone`; the tool must not auto-downgrade from a failed HF/Megatron reference path to standalone. Reports must label this mode as `standalone-smoke`, and downstream summaries must not imply it satisfies normal reference-mode precision alignment.

Run:

1. Loss range check.
2. Determinism check with two identical forward runs.
3. Gradient sanity check: finite and non-zero.
4. Optional short loss trend check.
5. Weight update check.

If checks 1, 2, 3, and 5 pass and `reference_type=standalone` was explicitly requested, return top-level `passed`; noisy short loss trend is recorded in `details.trainability_warning` rather than using a separate status. If standalone was not explicitly requested, return `human_needed` with `failure_gate="reference_unavailable"` instead of standalone `passed`.

## Reference-Patchset Migration Mode (`reference_type=reference_migration`)

Some models must be migrated from a known-working reference checkout into the current target tree. For these models, the KB entry sets `migration.required: true` and lists `migration.reference_root`, `migration.baseline_script`, and `migration.lossdiff_bundle`.

In reference-migration mode, the precision gate is a **same-batch** loss diff between the reference launcher and the target launcher, not HF vs Omni.

Required inputs:

- `baseline_script` — the launcher under `migration.reference_root`.
- `target_script` — the migrated launcher in the target tree.
- `lossdiff_bundle` — the lossdiff helper bundle declared in KB.
- `lite_checkpoint` — a lite checkpoint that exercises the full forward.

Run:

1. Run `baseline_script` for one iteration with the lite checkpoint, exporting the runtime batch tokens, runtime loss, and (when available) per-layer tensors with the lossdiff bundle.
2. Run `target_script` with the same exported batch and the same checkpoint, again exporting runtime loss and tensors.
3. Compare step-1 forward loss with `loss_mean_diff <= 1e-3`; rerun once with the same seed if a flake is suspected, then escalate.
4. Run the model's deterministic migration verifier (named in the source YAML's `validation.verifier_script`) and embed its `validator` block in the report under `details.migration_invariants`. A failure here is a hard fail of the loss-diff gate even if loss matches numerically — model-specific code in Megatron is a contract violation regardless of measured loss.
5. When loss diverges, prefer to bisect by replaying the exported batch through baseline and target, then comparing per-layer tensors via the lossdiff bundle, before suspecting kernel-level divergence.

Report this mode as `reference_type="reference_migration"` and include the baseline run's commit/branch info plus the verifier's `validator` block in `details`.

## Thresholds

| Metric | Threshold | Description |
|--------|-----------|-------------|
| loss diff | `< 1e-3` | `abs(loss_omni - loss_ref)` |
| tensor relative L1 | `< 1e-4` | Per-layer tensor relative L1 norm difference |
| train-step loss diff | `< 1e-3` | Per-step `abs(loss_omni_steps[i] - loss_ref_steps[i])` for each post-update step `i >= 2` |
| grad norm finite check | finite only | Grad norms are diagnostic except NaN/Inf, which fails the gate |

### Per-Layer Tensor Diagnosis (agent-driven)

When Stage 1b or Stage 2b is triggered, the agent must devise its own tensor comparison approach:
1. Load saved Omni tensors and reference tensors from their respective directories
2. For each matching layer tensor pair, compute: max absolute diff, relative L1 norm diff, cosine similarity
3. Identify the first diverging layer (relative L1 > 1e-4 threshold)
4. Write diagnosis summary to `$run_dir/tensor_compare_forward/diagnosis.json` or `$run_dir/tensor_compare_backward/diagnosis.json`

No external comparison scripts are required — the agent implements this logic inline using PyTorch tensor operations.

## Error Handling

| Error Scenario | Status | Retry | Description |
|----------------|--------|-------|-------------|
| Phase 2 not completed | `human_needed` | No retry | Prerequisite not met |
| `omni_script` missing or not executable | `human_needed` | No retry | Phase 3 must rebuild the script from the Phase 1 example |
| HF tensor dump runtime error | `failed` | Retry up to caller limit | May be environment issue or OOM |
| Omni script runtime error | `failed` | Retry up to caller limit | May be parameter, environment, or resource issue |
| Loss diff exceeds threshold | `human_needed` | No retry | Stage 1b diagnosis is required |
| Train-step loss diff exceeds threshold | `human_needed` | No retry | Stage 2b diagnosis is required |
| Grad norm is NaN/Inf | `human_needed` | No retry | Backward/update path diagnosis is required |
| Tensor diagnosis runtime error | `failed` | Retry up to caller limit | Possibly a tensor format mismatch or path error; escalate if still failing after retries |
