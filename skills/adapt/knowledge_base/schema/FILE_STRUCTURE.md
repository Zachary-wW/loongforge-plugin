# LoongForge Model File Structure

> This file is indexed by SKILL.md and defines the file layout, generation order, and key format requirements for three model types.

---

## Required Files for LLM

```
loongforge/models/foundation/<family>/
  ├── <family>_config.py
  ├── <family>_layer_spec.py
  ├── <family>_model.py
  └── __init__.py
configs/models/<family>/
  ├── <size>.yaml
  └── ckpt_convert/<family>_convert.yaml
examples/<family>/
  ├── pretrain/pretrain_<family>.sh
  ├── finetune/finetune_<family>.sh        <- SFT script (required, must include --chat-template <family>)
  └── checkpoint_convert/
      └── convert_<family>_hf_to_mcore.sh
(chat_template is registered in loongforge/data/chat_template.py, append _register_chat_template(name="<family>", ...) at the end of the file)
```

## Required Files for VLM

Additional files beyond LLM:

```
loongforge/models/encoder/<family>_vision_models/
  ├── <family>_config.py
  ├── <family>_layer_spec.py
  └── <family>_model.py
loongforge/data/multimodal/<family>_task_encoder.py
(convert shell must contain three segments: foundation + encoder + projector)
```

> **Note**: The `encoder/<family>_vision_models/` subdirectory does **not** need `__init__.py`. Encoder registration is done uniformly through the top-level `loongforge/models/encoder/__init__.py` (import the new encoder's config, layer_spec, and model classes in that top-level file).

## Required Files for Diffusion

```
loongforge/models/diffusion/<family>/
  ├── <family>_config.py     (inherits BaseModelStditConfig)
  ├── <family>_layer_spec.py
  ├── <family>_model.py
  └── __init__.py
```

## Required Registration Files

> **These three files already exist and do not need to be created; only append at the end. Missing any one of them will cause training startup failure.**

| File | LLM | VLM |
|------|-----|-----|
| `loongforge/utils/constants.py` | Append at the end of `LanguageModelFamilies`: `<FAMILY> = "<family>"` | Additionally append at the end of `VisionLanguageModelFamilies`: `<FAMILY> = "<family>"` |
| `loongforge/models/foundation/__init__.py` | Append two lines at the end of the file (see "Key File Formats -> foundation/__init__.py") | Same as LLM |
| `loongforge/models/encoder/__init__.py` | Do not modify | Append at the end of the file (see "Key File Formats -> encoder/__init__.py") |

## Generation Order (Strict)

New `.py` files created by `wrap_megatron` / `new_impl` must be generated **before** `_layer_spec.py` (layer_spec needs to import them):

```
loongforge/utils/constants.py  <- append family constant (LLM: LanguageModelFamilies; VLM: additionally append VisionLanguageModelFamilies)
  -> [new_impl / wrap_megatron new .py (in dependency order, e.g., gated_attention.py)]
    -> <family>_config.py
    -> <family>_layer_spec.py
      -> <family>_model.py          <- inject PHASE1_VERIFY hook
        -> <family>/__init__.py
          -> foundation/__init__.py  <- append AutoModel.register
            -> configs/models/<family>/<model>.yaml
              -> examples/<family>/pretrain/pretrain_<family>.sh
                -> examples/<family>/finetune/finetune_<family>.sh
                  -> [chat_template, if has_chat_template=true]
```

> `configs/models/<family>/ckpt_convert/<family>_convert.yaml` and `examples/<family>/checkpoint_convert/convert_<model>_hf_to_mcore.sh` are generated by **Phase 2**, not included in the Phase 1 generation order.

VLM path (when creating a new Encoder):

```
encoder/<enc>_vision_models/<enc>_config.py -> layer_spec -> model -> adapter
  -> configs/models/image_encoder/<enc>.yaml
  -> configs/models/<vlm_family>/<vlm_model>.yaml
  -> examples/<vlm_family>/pretrain + finetune sh
```

> Encoder / projector convert YAML and convert shell are generated by **Phase 2**.

## Key File Formats

### `<family>/__init__.py` (strictly two lines)

```python
from .<family>_model import <Family>Model
from .<family>_config import <Family>Config
```

### `foundation/__init__.py` Append Format (at the end of the file, do not modify other lines)

```python
from .<family> import <Family>Model, <Family>Config
AutoModel.register(<Family>Config, <Family>Model, exist_ok=True)
```

### `encoder/__init__.py` Append Format (VLM only, at the end of the file, do not modify other lines)

Append imports to the end of the import section at the top of the file (one line per component), and append AutoModel.register to the end of the register section at the bottom of the file:

```python
# Append at the end of the import section (one line per component):
from .<family>_vision_models.<family>_config import <Family>VisionConfig
from .<family>_vision_models.<family>_model import <Family>VisionModel
# If there is a standalone adapter, also append:
# from .<family>_vision_models.<family>_adapter import <Family>Adapter, <Family>AdapterConfig

# Append at the end of the register section:
AutoModel.register(<Family>VisionConfig, <Family>VisionModel)
# If there is an adapter:
# AutoModel.register(<Family>AdapterConfig, <Family>Adapter)
```

## Few-shot Reference Paths

| Scenario | Reference Path |
|----------|---------------|
| Dense LLM (simplest baseline) | `loongforge/models/foundation/llama/` |
| GQA + QKNorm + MoE | `loongforge/models/foundation/qwen3/` |
| MLA (DeepSeek family) | `loongforge/models/foundation/deepseek/` |
| Hybrid (Linear Attn + wrap_megatron + new_impl) | `loongforge/models/foundation/qwen3_next/` |
| Convert YAML | `configs/models/qwen3/ckpt_convert/qwen3_convert.yaml` |
| VLM three-segment convert | `examples/internvl3.5/checkpoint_convert/` |
