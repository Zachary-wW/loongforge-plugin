# Witness: deepseek_v4_flash (attention)

## Purpose

This witness freezes the `loongforge/models/foundation/deepseek_v4/deepseek_v4_attention.py`
generated by `/loongforge:adapt` (pre-perf-review-pipeline) as a regression
anchor. It is the source case that motivated the perf/parallel improvement plan.

It is used to:

1. **Validate** that newly added perf rules / scale signatures / parallel recipes
   actually catch the issues a real adapt run produced.
2. **Quantify** improvement across phased rollout (batch 1 → 4) via the
   `expected_*` files in this directory.
3. **Guard against regression**: any future change to perf-rules / reviewer that
   makes it stop catching one of the listed findings fails the witness check.

## Scope (batch 1)

`expected_perf_findings.yml` covers Phase 1 reviewer findings on the attention
file only. Whole-model probe expectations (memory peak, top tensors) are added
in batch 3 once `Step 7c` lands.

## Scope (out of scope)

- Numerical correctness regressions (covered by Phase 3 loss-diff, not perf).
- Cross-layer / cross-iter perf (PP bubbles, first-iter compile cost) — not
  addressable by single-layer probe; tracked in batch 4+.
- Algorithm-level kernel choice (FA3 vs cuDNN SDPA) — not a static-rule concern.
